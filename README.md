# Neural Network from Scratch — Three-Way Comparison

A coursework project comparing three feedforward neural network implementations
on two classification datasets (Iris, Breast Cancer Diagnostic):

1. **`Custom_nn.py`** — written from scratch in NumPy. Variable depth, sigmoid /
   ReLU / tanh activations, BCE loss with one-hot targets, optional class
   weights, Glorot init.
2. **`LLM_nn.py`** — a NumPy reference implementation generated by an LLM, kept
   intact for comparison. Softmax output, cross-entropy loss, He init for ReLU.
3. **`PyTorch_nn.py`** — a small `nn.Module` wrapper using `CrossEntropyLoss`
   and Adam / SGD.

The point of the project is *not* to beat PyTorch. It is to (a) verify that a
hand-written backprop matches the high-level frameworks on a real dataset and
(b) characterise how depth, width, activation, and learning rate affect a small
classifier.

## Quickstart

```bash
git clone <repo-url> nn_assignment
cd nn_assignment
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
jupyter notebook
```

Open either notebook and run top-to-bottom:

- `main.ipynb` — Iris (3-class), PyTorch end-to-end.
- `Breast_Cancer_Diagnostic.ipynb` — full CRISP-DM walkthrough on the breast
  cancer dataset, with grid search over the scratch model and head-to-head
  comparison against `LLM_nn` and `TorchModel`.

## Datasets

- **Iris** ships in `data/Iris.csv` (150 samples, 4 features, 3 classes).
- **Breast Cancer Diagnostic** is from the [TU Wien ML 2025 Kaggle
  competition](https://www.kaggle.com/competitions/184-702-tu-ml-2025-s-breast-cancer-diagnostic/data).
  Download the competition files and place them in
  `184-702-tu-ml-2025-s-breast-cancer-diagnostic/` at the repo root (this
  folder is gitignored). 569 samples, 30 features, binary class.

## Methodology notes

- **Train / val / test split.** The breast cancer notebook splits 80/20 into
  trainval/test, then carves a further 80/20 train/val out of trainval. Grid
  search picks the best config by validation accuracy; the final test metric is
  computed *once* on the held-out test set after model selection.
- **Class weights** are derived from inverse class frequency and applied
  consistently across the three implementations.
- **Reproducibility.** All three implementations accept a `seed` argument
  (`np.random.seed` / `torch.manual_seed`). Notebooks fix `random_state=42`.

## Caveats / things this project does *not* do

- The scratch backprop in `Custom_nn.py` assumes a sigmoid output layer
  (BCE-with-sigmoid simplification). An assertion enforces this.
- `LLM_nn.py` is intentionally left close to the original generated code so
  that the comparison reflects "what an LLM hands you" rather than a polished
  version.
- The three implementations differ in data layout (`(features, samples)` vs
  `(samples, features)`) and training regime (full-batch vs minibatch). This is
  documented but not unified — comparisons are within-implementation across
  hyperparameters.
